# Issue 5 - 優化 video-to-images 的卷動去重與 focus 預覽

本 issue 延續 [issue-4](../issue-4/plan.md) 已建立的 round-based viewer 與 thumbnail 工作流，聚焦處理 `video-to-images` 在人工判讀時的兩個實際痛點：

- 很多畫面是上下卷動，連續兩張圖雖不是完全相同，但資訊高度重疊，會增加人工檢查成本。
- `focus` 圖目前預覽放大程度不足，使用者在比對細節時仍需要額外放大或切換視角。

執行結果請寫入同目錄的 [impl.md](./impl.md)。

## 1. 背景

目前 `video-to-images` 已能把錄影切成代表圖，協助 `VIDEO / HAR / Recording` 三方比對，這條路線在 POC 階段已證明是可行的輔助工具。

但在實際檢查過程中，仍有兩個會直接拉高人工成本的問題：

1. 上下卷動造成的相鄰畫面，常常只是同一頁的上下位移，資訊重複度高。
2. 焦點圖雖可預覽，但尺寸仍不足以快速辨識細節差異。

這兩個問題都直接影響「能不能更快找到可用驗證畫面」這個 POC 主目標，因此值得先做規則型優化，再判斷是否值得後續工程化。

## 2. 目標

本 issue 要回答：

1. 若對連續畫面加入 `50% overlap` 略過規則，是否能有效降低卷動畫面的重複輸出，同時不明顯損失可判讀性。
2. 若 `focus` 圖預覽再放大兩級，是否能提升人工比對效率，且不破壞目前 viewer 的使用流暢度。

## 3. 執行項目

### Task A. 建立卷動畫面 overlap 略過規則

- Goal: 針對上下卷動造成的高重疊連續畫面，加入明確的略過規則，降低人工判讀負擔。
- 規則草案：
  - 若上一張與下一張圖的畫面內容具有 `50%` 以上 overlap，則略過下一張，不作為主要代表圖。
  - 此規則優先用於上下卷動造成的連續畫面，並需保留限制說明，避免誤把不同狀態頁誤判為同頁延伸。
- 預期輸出：
  - overlap 判斷規則與門檻定義
  - 套用規則後的代表圖輸出結果或統計差異
  - 限制與誤判風險說明

### Task B. 調整 focus 圖預覽大小

- Goal: 在 viewer 中，當圖片進入 `focus` 狀態時，預覽尺寸再放大兩級，讓人工檢查更容易辨識細節。
- 預期輸出：
  - `focus` 狀態的尺寸規則調整
  - 放大前後的畫面差異說明
  - 對 layout / 操作體驗的影響評估

### Task C. 驗證對人工判讀流程的影響

- Goal: 確認這兩項優化是否真的有助於支撐驗證用途，而不是只改 UI 外觀。
- 預期輸出：
  - 一組可比較的樣本或觀察紀錄
  - 哪些情境有效、哪些情境仍需人工補強
  - feasible / partial / infeasible 判斷

### Task D. 文件同步

- Goal: 更新 issue 紀錄與相關文件，讓下一輪能直接承接這次調整與限制。
- 預期輸出：
  - `issue/issue-5/impl.md`
  - 視需要更新 `README.md`
  - 視需要更新 `tools/video-to-images/README.md`

## 5. 成功標準

本 issue 若要算成功，至少需滿足：

1. 已明確定義 `50% overlap` 略過規則，並有實際樣本或輸出結果支撐。
2. 能說明該規則對上下卷動畫面的幫助，以及可能誤判的限制。
3. `focus` 圖預覽已再放大兩級，且可清楚描述調整結果。
4. 能回答這兩項優化是否真的降低人工判讀成本。
5. `impl.md` 已完整補上 FEEDBACK。

## 6. 風險

- `50% overlap` 屬於 heuristic，可能把局部內容改變但版型相似的畫面誤判成可略過。
- 若卷動速度快、動畫多或有 sticky header，overlap 判斷可能不穩定。
- `focus` 預覽放大後，可能壓縮其他區塊可視範圍，需留意 viewer 版面是否失衡。
- 即使重複畫面減少，關鍵頁面辨識仍可能需要人工標註，不能誤認為已完成全自動判讀。

## 7. 驗收重點

本 issue 驗收重點是：

- 卷動畫面是否真的因 overlap 規則而更精簡
- `focus` 預覽放大後是否更利於人工比對
- 規則與限制是否已寫清楚，足以支撐下一輪繼續驗證
